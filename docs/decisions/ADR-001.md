# ADR-001: Phase 1 Architecture and Technology Decisions

## Status

Accepted

## Context

AegisVault Phase 1 needs a minimal but secure scaffold to validate the Inbox → classify → encrypt → Vault pipeline.

## Decisions

1. **Orchestration framework**: Self-built `aegis-agent` using Pydantic + asyncio. Rationale: minimal supply-chain attack surface, explicit state machine, no cloud dependencies.
2. **UI stack**: PyQt6 for Phase 1 tray + dialog. Rationale: same language as backend, fast iteration. Re-evaluate Tauri in Phase 3.
3. **Model service**: OpenAI-compatible abstraction over Ollama / LM Studio / llama.cpp server. Rationale: unified interface, switchable backend.
4. **Password vault**: KeePassXC-cli as primary, pass as optional. Rationale: cross-platform, local, supports hardware keys.
5. **Vector retrieval**: Deferred to Phase 3. Phase 1 uses SQLite FTS on tags/path/time. Rationale: lower complexity and attack surface.
6. **Sandbox**: Phase 1 uses dedicated user account + firewall outbound block. Phase 2 adds bubblewrap/AppContainer.
7. **Telemetry**: Zero automatic telemetry. Local encrypted logs only, manual export optional.

## Consequences

- Pros: smaller dependency tree, faster security audit, clear migration path.
- Cons: more code to maintain than using LangChain; PyQt6 may be replaced later.
