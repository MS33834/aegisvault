# ADR-002: Platform Connection Management

## Status

Accepted

## Context

用户要求 AegisVault 提供统一的平台连接管理界面，支持：
- 本地模型服务（LM Studio / Ollama / llama.cpp server）
- 云端模型服务（OpenAI / Anthropic / 自定义 OpenAI 兼容端点）
- 可自定义新增平台
- 机器环境为 Windows 11

## Decision

1. **引入独立的 Platform 层**：所有外部/本地服务连接统一抽象为 `Connection`，由 `ConnectionManager` 管理。
2. **默认本地优先**：未明确配置时，系统仅使用本地 `http://127.0.0.1` 端点，且核心 Vault 加密/解密不依赖任何外部平台。
3. **云连接需显式授权**：用户必须在 UI 中勾选“允许此连接访问云端”，且云连接仅用于非敏感任务（如通用分类、摘要）。
4. **敏感操作强制离线**：涉及加密密钥、密码填充、Vault 内容检索时，必须仅使用标记为 `is_local=true` 且监听 `127.0.0.1` 的连接；否则拒绝执行。
5. **Windows 11 适配**：
   - 主控密钥默认使用 Windows DPAPI 保护（本地用户账户绑定）。
   - 可选 Windows Hello 作为解锁方式（Phase 2）。
   - 防火墙规则使用 PowerShell `New-NetFirewallRule` 阻止核心进程出站。

## Consequences

- 优点：扩展性强，用户可按需接入不同模型服务；UI 统一。
- 风险：云连接增加泄露面。通过 UI 强制标识 + 代码层敏感任务拦截来降低风险。
